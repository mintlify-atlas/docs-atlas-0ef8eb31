## What is Motion GPU?

Motion GPU is a focused runtime for **fullscreen fragment shader pipelines**. Unlike general-purpose 3D engines, it does one thing well: running WGSL shaders on the full canvas using WebGPU.

Typical use cases include:

- Shadertoy-style GPU experiments and live-coding tools
- Generative art and procedural textures
- Multi-pass post-processing pipelines
- GPU particle simulations
- Shader editors and interactive visual experiments

<Note>
  Motion GPU requires a browser or runtime with WebGPU support and a **secure context** (`https://` or `localhost`).
</Note>

## Core workflow

<Steps>
  <Step title="Define a material">
    Use `defineMaterial()` to declare your WGSL fragment shader along with uniforms, textures, and storage buffers. Materials are immutable and validated at definition time.
  </Step>
  <Step title="Render with FragCanvas">
    Mount `<FragCanvas material={material} />` in your component tree. It manages the WebGPU context, pipeline, and render loop automatically.
  </Step>
  <Step title="Drive updates with useFrame">
    Use `useFrame()` inside the `<FragCanvas>` subtree to update uniforms, textures, and storage buffers every frame.
  </Step>
</Steps>

```typescript
import { FragCanvas, defineMaterial, useFrame } from '@motion-core/motion-gpu/react';

const material = defineMaterial({
  fragment: `
fn frag(uv: vec2f) -> vec4f {
  let wave = 0.5 + 0.5 * sin(motiongpuUniforms.uTime + uv.x * 8.0);
  return vec4f(vec3f(wave), 1.0);
}
`,
  uniforms: { uTime: 0 }
});

function Runtime() {
  useFrame((state) => {
    state.setUniform('uTime', state.time);
  });
  return null;
}

export function App() {
  return (
    <div style={{ width: '100vw', height: '100vh' }}>
      <FragCanvas material={material}>
        <Runtime />
      </FragCanvas>
    </div>
  );
}
```
