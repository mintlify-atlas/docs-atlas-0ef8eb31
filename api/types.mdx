This page documents every public TypeScript type exported from `@motion-core/motion-gpu`. Types are organized by domain. All definitions are drawn directly from the runtime source.

## Material types

### `FragMaterialInput`

The raw input object accepted by [`defineMaterial`](/api/define-material). All fields except `fragment` are optional.

```typescript
interface FragMaterialInput<
  TUniformKey extends string = string,
  TTextureKey extends string = string,
  TDefineKey extends string = string,
  TIncludeKey extends string = string,
  TStorageBufferKey extends string = string
> {
  fragment: string;
  uniforms?: UniformMap<TUniformKey>;
  textures?: TextureDefinitionMap<TTextureKey>;
  defines?: MaterialDefines<TDefineKey>;
  includes?: MaterialIncludes<TIncludeKey>;
  storageBuffers?: StorageBufferDefinitionMap<TStorageBufferKey>;
}
```

<ResponseField name="fragment" type="string" required>
  WGSL source string. Must contain an entrypoint with the exact signature `fn frag(uv: vec2f) -> vec4f`.
</ResponseField>

<ResponseField name="uniforms" type="UniformMap<TUniformKey>">
  Initial uniform values keyed by WGSL identifier names.
</ResponseField>

<ResponseField name="textures" type="TextureDefinitionMap<TTextureKey>">
  Texture slot definitions keyed by texture uniform names.
</ResponseField>

<ResponseField name="defines" type="MaterialDefines<TDefineKey>">
  Compile-time constants injected as WGSL `const` declarations before the fragment shader.
</ResponseField>

<ResponseField name="includes" type="MaterialIncludes<TIncludeKey>">
  WGSL code chunks resolved by `#include <name>` directives in the fragment shader.
</ResponseField>

<ResponseField name="storageBuffers" type="StorageBufferDefinitionMap<TStorageBufferKey>">
  Storage buffer definitions for compute shader workflows.
</ResponseField>

### `FragMaterial`

The frozen, immutable material object returned by `defineMaterial`. Safe to share across components and renders.

```typescript
interface FragMaterial<
  TUniformKey extends string = string,
  TTextureKey extends string = string,
  TDefineKey extends string = string,
  TIncludeKey extends string = string,
  TStorageBufferKey extends string = string
> {
  readonly fragment: string;
  readonly uniforms: Readonly<UniformMap<TUniformKey>>;
  readonly textures: Readonly<TextureDefinitionMap<TTextureKey>>;
  readonly defines: Readonly<MaterialDefines<TDefineKey>>;
  readonly includes: Readonly<MaterialIncludes<TIncludeKey>>;
  readonly storageBuffers: Readonly<StorageBufferDefinitionMap<TStorageBufferKey>>;
}
```

<Note>
  All fields are deeply frozen. Do not attempt to mutate a `FragMaterial` — always create a new one with `defineMaterial`.
</Note>

<ResponseField name="fragment" type="string">
  Original WGSL source string as supplied to `defineMaterial`.
</ResponseField>

<ResponseField name="uniforms" type="Readonly<UniformMap<TUniformKey>>">
  Frozen uniform value map. Values are cloned during material creation.
</ResponseField>

<ResponseField name="textures" type="Readonly<TextureDefinitionMap<TTextureKey>>">
  Frozen texture definition map.
</ResponseField>

<ResponseField name="defines" type="Readonly<MaterialDefines<TDefineKey>>">
  Frozen define map used for WGSL const injection.
</ResponseField>

<ResponseField name="includes" type="Readonly<MaterialIncludes<TIncludeKey>>">
  Frozen include source map.
</ResponseField>

<ResponseField name="storageBuffers" type="Readonly<StorageBufferDefinitionMap<TStorageBufferKey>>">
  Frozen storage buffer definition map.
</ResponseField>

### `ResolvedMaterial`

A fully resolved, renderer-ready snapshot derived from a `FragMaterial`. Produced internally by `resolveMaterial` and cached per material instance.

```typescript
interface ResolvedMaterial<
  TUniformKey extends string = string,
  TTextureKey extends string = string,
  TIncludeKey extends string = string,
  TStorageBufferKey extends string = string
> {
  fragmentWgsl: string;
  fragmentLineMap: MaterialLineMap;
  uniforms: UniformMap<TUniformKey>;
  textures: TextureDefinitionMap<TTextureKey>;
  uniformLayout: UniformLayout;
  textureKeys: TTextureKey[];
  signature: string;
  fragmentSource: string;
  includeSources: MaterialIncludes<TIncludeKey>;
  defineBlockSource: string;
  source: Readonly<MaterialSourceMetadata> | null;
  storageBufferKeys: TStorageBufferKey[];
  storageTextureKeys: TTextureKey[];
}
```

<ResponseField name="fragmentWgsl" type="string">
  Final WGSL fragment source after define injection and include expansion.
</ResponseField>

<ResponseField name="fragmentLineMap" type="MaterialLineMap">
  1-based mapping from generated fragment lines back to user source lines. Used for shader error diagnostics.
</ResponseField>

<ResponseField name="uniforms" type="UniformMap<TUniformKey>">
  Cloned uniform values from the source material.
</ResponseField>

<ResponseField name="textures" type="TextureDefinitionMap<TTextureKey>">
  Cloned texture definitions from the source material.
</ResponseField>

<ResponseField name="uniformLayout" type="UniformLayout">
  Packed GPU uniform buffer layout resolved using WGSL alignment rules.
</ResponseField>

<ResponseField name="textureKeys" type="TTextureKey[]">
  Sorted array of texture keys derived from the texture definitions map.
</ResponseField>

<ResponseField name="signature" type="string">
  Deterministic JSON signature used for renderer cache invalidation. Changes when shader, uniforms, textures, or storage buffers change.
</ResponseField>

<ResponseField name="fragmentSource" type="string">
  Original user fragment source before preprocessing.
</ResponseField>

<ResponseField name="includeSources" type="MaterialIncludes<TIncludeKey>">
  Normalized include source map.
</ResponseField>

<ResponseField name="defineBlockSource" type="string">
  Deterministic define block WGSL used for diagnostics source mapping.
</ResponseField>

<ResponseField name="source" type="Readonly<MaterialSourceMetadata> | null">
  Optional callsite metadata captured at `defineMaterial` time, used for diagnostics.
</ResponseField>

<ResponseField name="storageBufferKeys" type="TStorageBufferKey[]">
  Sorted array of storage buffer keys. Empty when no storage buffers are declared.
</ResponseField>

<ResponseField name="storageTextureKeys" type="TTextureKey[]">
  Sorted array of texture keys where `storage: true`. Empty when no storage textures are declared.
</ResponseField>

### `MaterialDefineValue`

Allowed value types for WGSL `const` define injection.

```typescript
type MaterialDefineValue = boolean | number | TypedMaterialDefineValue;
```

Plain `boolean` and `number` values are inferred as WGSL `bool`, `f32`, or `i32`. Use `TypedMaterialDefineValue` for explicit type control.

---

### `TypedMaterialDefineValue`

Explicitly typed compile-time define declaration with a WGSL type tag.

```typescript
type TypedMaterialDefineValue =
  | { type: 'bool'; value: boolean }
  | { type: 'f32' | 'i32' | 'u32'; value: number };
```

<ResponseField name="type" type="'bool' | 'f32' | 'i32' | 'u32'" required>
  WGSL scalar type for the constant.
</ResponseField>

<ResponseField name="value" type="boolean | number" required>
  Literal value matching the selected WGSL type.
</ResponseField>

---

### `MaterialDefines`

Define map keyed by uniform-compatible identifier names.

```typescript
type MaterialDefines<TKey extends string = string> = Record<TKey, MaterialDefineValue>;
```

---

### `MaterialIncludes`

Include map keyed by the identifier used in `#include <name>` directives.

```typescript
type MaterialIncludes<TKey extends string = string> = Record<TKey, string>;
```

## Uniform types

### `UniformType`

WGSL-compatible uniform primitive and aggregate types supported by MotionGPU.

```typescript
type UniformType = 'f32' | 'vec2f' | 'vec3f' | 'vec4f' | 'mat4x4f';
```

| Value | WGSL type | Runtime shape |
|-------|-----------|---------------|
| `'f32'` | `f32` | `number` |
| `'vec2f'` | `vec2f` | `[number, number]` |
| `'vec3f'` | `vec3f` | `[number, number, number]` |
| `'vec4f'` | `vec4f` | `[number, number, number, number]` |
| `'mat4x4f'` | `mat4x4f` | `number[] \| Float32Array` |

---

### `UniformValue`

All accepted input shapes for a single uniform value.

```typescript
type UniformValue =
  | number
  | [number, number]
  | [number, number, number]
  | [number, number, number, number]
  | TypedUniform<'f32'>
  | TypedUniform<'vec2f'>
  | TypedUniform<'vec3f'>
  | TypedUniform<'vec4f'>
  | TypedUniform<'mat4x4f'>;
```

Bare number literals and tuples are accepted alongside explicitly typed `TypedUniform` objects. MotionGPU infers the WGSL type from the runtime shape.

---

### `TypedUniform`

An explicitly typed uniform declaration pairing a WGSL type tag with its runtime value.

```typescript
interface TypedUniform<
  TType extends UniformType = UniformType,
  TValue extends UniformValueByType[TType] = UniformValueByType[TType]
> {
  type: TType;
  value: TValue;
}
```

<ResponseField name="type" type="UniformType" required>
  WGSL type tag. Determines how `value` is packed into the GPU buffer.
</ResponseField>

<ResponseField name="value" type="UniformValueByType[TType]" required>
  Runtime value matching the declared `type`. See `UniformValue` for per-type shapes.
</ResponseField>

---

### `UniformMat4Value`

Accepted value formats for `mat4x4f` uniforms.

```typescript
type UniformMat4Value = number[] | Float32Array;
```

---

### `UniformMap`

Uniform map keyed by WGSL identifier names.

```typescript
type UniformMap<TKey extends string = string> = Record<TKey, UniformValue>;
```

### `UniformLayout`

GPU uniform buffer layout resolved from a `UniformMap` using WGSL std140/std430 alignment rules.

```typescript
interface UniformLayout {
  entries: UniformLayoutEntry[];
  byName: Record<string, UniformLayoutEntry>;
  byteLength: number;
}
```

<ResponseField name="entries" type="UniformLayoutEntry[]">
  Layout entries sorted by uniform name.
</ResponseField>

<ResponseField name="byName" type="Record<string, UniformLayoutEntry>">
  Fast lookup table keyed by uniform name.
</ResponseField>

<ResponseField name="byteLength" type="number">
  Total uniform buffer size in bytes after alignment padding.
</ResponseField>

---

### `UniformLayoutEntry`

Resolved metadata for a single uniform field inside the packed uniform buffer.

```typescript
interface UniformLayoutEntry {
  name: string;
  type: UniformType;
  offset: number;
  size: number;
}
```

<ResponseField name="name" type="string">
  Uniform field name matching the WGSL identifier.
</ResponseField>

<ResponseField name="type" type="UniformType">
  WGSL type tag for this field.
</ResponseField>

<ResponseField name="offset" type="number">
  Byte offset of this field within the packed uniform buffer.
</ResponseField>

<ResponseField name="size" type="number">
  Field byte size excluding trailing alignment padding.
</ResponseField>

## Texture types

### `TextureDefinition`

Per-texture sampling and upload configuration declared in a material.

```typescript
interface TextureDefinition {
  source?: TextureValue;
  colorSpace?: 'srgb' | 'linear';
  flipY?: boolean;
  generateMipmaps?: boolean;
  premultipliedAlpha?: boolean;
  update?: TextureUpdateMode;
  anisotropy?: number;
  filter?: GPUFilterMode;
  addressModeU?: GPUAddressMode;
  addressModeV?: GPUAddressMode;
  storage?: boolean;
  format?: GPUTextureFormat;
  width?: number;
  height?: number;
  fragmentVisible?: boolean;
}
```

<ResponseField name="source" type="TextureValue">
  Default or initial texture value for this slot.
</ResponseField>

<ResponseField name="colorSpace" type="'srgb' | 'linear'">
  Source color space used for GPU format and decode decisions. Defaults to `'srgb'`.
</ResponseField>

<ResponseField name="flipY" type="boolean">
  Flips the texture vertically during upload. Defaults to `false`.
</ResponseField>

<ResponseField name="generateMipmaps" type="boolean">
  Enables mipmap generation after upload. Defaults to `false`.
</ResponseField>

<ResponseField name="premultipliedAlpha" type="boolean">
  Enables premultiplied-alpha upload mode. Defaults to `false`.
</ResponseField>

<ResponseField name="update" type="TextureUpdateMode">
  Dynamic source update strategy. See `TextureUpdateMode`.
</ResponseField>

<ResponseField name="anisotropy" type="number">
  Sampler anisotropy level. Clamped to device limits internally.
</ResponseField>

<ResponseField name="filter" type="GPUFilterMode">
  Min/mag filter mode for the sampler (`'nearest'` or `'linear'`).
</ResponseField>

<ResponseField name="addressModeU" type="GPUAddressMode">
  U-axis texture address mode (`'clamp-to-edge'`, `'repeat'`, `'mirror-repeat'`).
</ResponseField>

<ResponseField name="addressModeV" type="GPUAddressMode">
  V-axis texture address mode.
</ResponseField>

<ResponseField name="storage" type="boolean">
  When `true`, the texture is also writable by compute passes as a storage texture. Requires `format`.
</ResponseField>

<ResponseField name="format" type="GPUTextureFormat">
  Texture format override. Required when `storage` is `true`. Must be a storage-compatible format.
</ResponseField>

<ResponseField name="width" type="number">
  Explicit texture width in pixels. Required for storage textures without a source.
</ResponseField>

<ResponseField name="height" type="number">
  Explicit texture height in pixels. Required for storage textures without a source.
</ResponseField>

<ResponseField name="fragmentVisible" type="boolean">
  When `true` (default), the texture is sampled in the fragment shader. Set to `false` for compute-only textures.
</ResponseField>

### `TextureValue`

Union of all accepted texture input shapes.

```typescript
type TextureValue = TextureData | TextureSource | null;
```

Pass `null` to clear a texture slot at runtime.

---

### `TextureMap`

Runtime texture value map keyed by texture uniform names.

```typescript
type TextureMap<TKey extends string = string> = Record<TKey, TextureValue>;
```

---

### `TextureUpdateMode`

Update strategy controlling when a dynamic texture source is re-uploaded to the GPU.

```typescript
type TextureUpdateMode = 'once' | 'onInvalidate' | 'perFrame';
```

| Value | Behavior |
|-------|----------|
| `'once'` | Upload once on first use. Never re-uploaded. |
| `'onInvalidate'` | Re-upload when the frame is invalidated. |
| `'perFrame'` | Re-upload every frame. Use for video or canvas sources. |

## Storage buffer types

### `StorageBufferDefinition`

Definition of a single GPU storage buffer resource.

```typescript
interface StorageBufferDefinition {
  size: number;
  type: StorageBufferType;
  access?: StorageBufferAccess;
  initialData?: Float32Array | Uint32Array | Int32Array;
}
```

<ResponseField name="size" type="number" required>
  Buffer size in bytes. Must be greater than 0 and a multiple of 4.
</ResponseField>

<ResponseField name="type" type="StorageBufferType" required>
  WGSL type annotation used for shader binding codegen.
</ResponseField>

<ResponseField name="access" type="StorageBufferAccess">
  Access mode in compute shaders. Defaults to `'read-write'`.
</ResponseField>

<ResponseField name="initialData" type="Float32Array | Uint32Array | Int32Array">
  Optional initial data uploaded to the buffer on creation. Cloned during material creation.
</ResponseField>

---

### `StorageBufferType`

All supported WGSL element types for storage buffers.

```typescript
type StorageBufferType =
  | 'array<f32>'
  | 'array<vec2f>'
  | 'array<vec3f>'
  | 'array<vec4f>'
  | 'array<u32>'
  | 'array<i32>'
  | 'array<vec4u>'
  | 'array<vec4i>';
```

---

### `StorageBufferAccess`

Access mode controlling read/write permissions in compute shaders.

```typescript
type StorageBufferAccess = 'read' | 'read-write';
```

Use `'read'` for buffers that are only consumed by shaders. Use `'read-write'` (default) for buffers that shaders also write to.

---

### `StorageBufferDefinitionMap`

Named storage buffer definition map.

```typescript
type StorageBufferDefinitionMap<TKey extends string = string> = Record<TKey, StorageBufferDefinition>;
```

## Render pass types

### `RenderPass`

Formal render pass contract used by the MotionGPU render graph. Extends `RenderPassFlags`.

```typescript
interface RenderPass extends RenderPassFlags {
  enabled?: boolean;
  needsSwap?: boolean;
  input?: RenderPassInputSlot;
  output?: RenderPassOutputSlot;
  setSize?: (width: number, height: number) => void;
  render: (context: RenderPassContext) => void;
  dispose?: () => void;
}
```

<ResponseField name="enabled" type="boolean">
  Enables or disables this pass without removing it from the render graph. Defaults to `true`.
</ResponseField>

<ResponseField name="needsSwap" type="boolean">
  When `true`, triggers a source/target ping-pong buffer swap after this pass renders.
</ResponseField>

<ResponseField name="input" type="RenderPassInputSlot">
  Input slot consumed by this pass. Defaults to `'source'`.
</ResponseField>

<ResponseField name="output" type="RenderPassOutputSlot">
  Output slot written by this pass. Defaults to `'target'`.
</ResponseField>

<ResponseField name="setSize" type="(width: number, height: number) => void">
  Called by the runtime when the canvas size or DPR changes.
</ResponseField>

<ResponseField name="render" type="(context: RenderPassContext) => void" required>
  Called each frame to execute pass commands. Receives a `RenderPassContext`.
</ResponseField>

<ResponseField name="dispose" type="() => void">
  Called when the renderer is destroyed. Release pass-owned GPU resources here.
</ResponseField>

---

### `RenderPassFlags`

Per-pass render attachment flags.

```typescript
interface RenderPassFlags {
  clear?: boolean;
  clearColor?: [number, number, number, number];
  preserve?: boolean;
}
```

<ResponseField name="clear" type="boolean">
  Clears the output attachment before drawing. Defaults to `false`.
</ResponseField>

<ResponseField name="clearColor" type="[number, number, number, number]">
  RGBA clear color used when `clear` is `true`. Values in `[0, 1]` range.
</ResponseField>

<ResponseField name="preserve" type="boolean">
  Stores the output attachment contents after the pass completes. Defaults to `true`.
</ResponseField>

---

### `RenderPassContext`

Execution context passed to `RenderPass.render`. Extends `Required<RenderPassFlags>`.

```typescript
interface RenderPassContext extends Required<RenderPassFlags> {

  width: number;
  height: number;

    view?: GPUTextureView;
    clear?: boolean;
    clearColor?: [number, number, number, number];
    preserve?: boolean;
  }) => GPURenderPassEncoder;
}
```

<ResponseField name="device" type="GPUDevice">
  The active WebGPU device for this frame.
</ResponseField>

<ResponseField name="commandEncoder" type="GPUCommandEncoder">
  Shared command encoder for the current frame. Submit commands through this encoder.
</ResponseField>

<ResponseField name="source" type="RenderTarget">
  Current source ping-pong surface.
</ResponseField>

<ResponseField name="target" type="RenderTarget">
  Current target ping-pong surface.
</ResponseField>

<ResponseField name="canvas" type="RenderTarget">
  Current frame canvas surface used for final presentation.
</ResponseField>

<ResponseField name="input" type="RenderTarget">
  Resolved input surface for this pass (based on `RenderPass.input`).
</ResponseField>

<ResponseField name="output" type="RenderTarget">
  Resolved output surface for this pass (based on `RenderPass.output`).
</ResponseField>

<ResponseField name="targets" type="Readonly<Record<string, RenderTarget>>">
  Snapshot of all named runtime render targets for this frame.
</ResponseField>

<ResponseField name="time" type="number">
  Elapsed time in seconds since the renderer started.
</ResponseField>

<ResponseField name="delta" type="number">
  Frame delta time in seconds.
</ResponseField>

<ResponseField name="width" type="number">
  Frame width in pixels (canvas width × DPR).
</ResponseField>

<ResponseField name="height" type="number">
  Frame height in pixels (canvas height × DPR).
</ResponseField>

<ResponseField name="beginRenderPass" type="function">
  Convenience method to begin a color render pass targeting the resolved output (or a provided view). Returns a `GPURenderPassEncoder`.
</ResponseField>

### `RenderPassInputSlot`

Input slot identifier used by render passes.

```typescript
type RenderPassInputSlot = 'source' | 'target' | RenderPassNamedSlot;
```

`'source'` and `'target'` are built-in ping-pong slots. Any other string resolves to a named render target key.

---

### `RenderPassOutputSlot`

Output slot identifier used by render passes.

```typescript
type RenderPassOutputSlot = 'source' | 'target' | 'canvas' | RenderPassNamedSlot;
```

`'canvas'` writes directly to the canvas surface. `'source'` and `'target'` write to ping-pong slots.

---

### `ComputePassLike`

Minimal interface for compute passes in the render graph. Compute passes do not participate in slot routing.

```typescript
interface ComputePassLike {
  readonly isCompute: true;
  enabled?: boolean;
  setSize?: (width: number, height: number) => void;
  dispose?: () => void;
}
```

<ResponseField name="isCompute" type="true" required>
  Discriminant flag identifying this as a compute pass. Must be `true`.
</ResponseField>

<ResponseField name="enabled" type="boolean">
  Enables or disables this pass without removing it from the graph.
</ResponseField>

<ResponseField name="setSize" type="(width: number, height: number) => void">
  Called by the runtime on canvas resize events.
</ResponseField>

<ResponseField name="dispose" type="() => void">
  Called when the renderer is destroyed. Release compute-owned GPU resources here.
</ResponseField>

---

### `AnyPass`

Union type for all pass types accepted by the render graph.

```typescript
type AnyPass = RenderPass | ComputePassLike;
```

## Frame & scheduler types

### `FrameState`

Mutable per-frame state object passed to all `useFrame` callbacks.

```typescript
interface FrameState {
  time: number;
  delta: number;
  setUniform: (name: string, value: UniformValue) => void;
  setTexture: (name: string, value: TextureValue) => void;
  writeStorageBuffer: (name: string, data: ArrayBufferView, options?: { offset?: number }) => void;
  readStorageBuffer: (name: string) => Promise<ArrayBuffer>;
  invalidate: (token?: FrameInvalidationToken) => void;
  advance: () => void;
  renderMode: RenderMode;
  autoRender: boolean;
  canvas: HTMLCanvasElement;
}
```

<ResponseField name="time" type="number">
  Elapsed time in seconds since the renderer started.
</ResponseField>

<ResponseField name="delta" type="number">
  Frame delta time in seconds. Clamped to `maxDelta` from the scheduler.
</ResponseField>

<ResponseField name="setUniform" type="(name: string, value: UniformValue) => void">
  Sets a uniform value for the current or next frame submission.
</ResponseField>

<ResponseField name="setTexture" type="(name: string, value: TextureValue) => void">
  Sets a texture slot value for the current or next frame submission.
</ResponseField>

<ResponseField name="writeStorageBuffer" type="(name: string, data: ArrayBufferView, options?: { offset?: number }) => void">
